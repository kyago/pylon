# Architect Analysis

파이프라인의 아키텍처 분석 단계를 단독으로 실행합니다.

## 입력
- `$ARGUMENTS`: 요구사항 또는 파이프라인 디렉토리 경로

## 실행

1. 요구사항을 분석합니다
2. 코드베이스를 탐색하여 영향 받는 영역을 파악합니다
3. 다음을 포함하는 `architecture.md`를 작성합니다:
   - 영향 받는 파일 목록
   - 변경 사항 설명
   - 새로 생성할 파일
   - 의존성 분석
   - 리스크 평가

파이프라인 디렉토리가 지정된 경우 `$PIPELINE_DIR/architecture.md`에 저장합니다.
